# Demo IoT Project with Docker
🌐 IoT Data Pipeline Architecture (ภาพรวมของโปรเจกต์)โปรเจกต์นี้คือ โครงสร้างพื้นฐานสำหรับระบบ IoT (IoT Infrastructure Stack) แบบครบวงจร 
ที่ถูกสร้างขึ้นมาเพื่อรับมือกับวัฏจักรของข้อมูล (Data Lifecycle) ตั้งแต่การรับข้อมูลจากฮาร์ดแวร์เซ็นเซอร์, การประมวลผลและคัดกรองข้อมูล, การจัดเก็บข้อมูลสถิติระยะยาว ไปจนถึงการแสดงผลข้อมูลออกมาเป็นกราฟที่เข้าใจง่าย
ระบบทั้งหมดถูกออกแบบให้อยู่ในรูปแบบ Container เพื่อให้ง่ายต่อการติดตั้ง โยกย้าย และขยายขนาด (Scalability) ในอนาคต🔄 

สถาปัตยกรรมการไหลของข้อมูล (Data Flow Architecture)ระบบถูกออกแบบการทำงานให้ไหลไปในทิศทางเดียวกัน (Pipeline) โดยมีลำดับขั้นตอนดังนี้:

[ IoT Devices / Sensors ] 
       │
       │ (ส่งข้อมูลผ่านโปรโตคอล MQTT)
       ▼
[ 📡 Mosquitto MQTT Broker ]  <-- ทำหน้าที่เป็นไปรษณีย์กลางคอยรับ-ส่งข้อความ
       │
       │ (Subscribe ข้อมูลที่ต้องการ)
       ▼
[ ⚙️ Node-RED ]               <-- ทำหน้าที่ประมวลผล, แปลงข้อมูล (Data Transformation) และตั้งเงื่อนไข
       │
       │ (ส่งข้อมูลผ่าน HTTP/API)
       ▼
[ 🗄️ InfluxDB ]               <-- ทำหน้าที่จัดเก็บข้อมูลผูกกับเวลา (Time-Series Database)

       │ (ดึงข้อมูลด้วยภาษา Flux Query)
       ▼
[ 📊 Grafana ]                <-- ทำหน้าที่แสดงผลเป็น Dashboard และแจ้งเตือน (Alerts)


🛠️ เครื่องมือที่ใช้งาน (Tools & Technologies)ระบบนี้ขับเคลื่อนด้วย 5 เครื่องมือหลักระดับอุตสาหกรรม (Industrial Standard) ได้แก่:

1. Docker & Docker Compose (Containerization)
คืออะไร: เครื่องมือจำลองสภาพแวดล้อมจำเพาะสำหรับรันโปรแกรม
บทบาทในโปรเจกต์: เป็นฐานรากที่ครอบทุก Service เอาไว้ ช่วยจัดการ Network ภายในให้ระบบทั้งหมดคุยกันได้ และแก้ปัญหา "รันบนเครื่องนี้ได้ แต่รันเครื่องอื่นไม่ได้"

2. Eclipse Mosquitto (MQTT Broker)
คืออะไร: เซิร์ฟเวอร์ตัวกลางสำหรับจัดการข้อความผ่านโปรโตคอล MQTT
บทบาทในโปรเจกต์: ทำหน้าที่เป็นศูนย์กลางรับข้อมูลจากเซ็นเซอร์นับร้อย/พันตัว (Publish) และกระจายข้อมูลนั้นไปยังแอปพลิเคชันที่รอรับอยู่ (Subscribe) อย่างรวดเร็วและใช้แบนด์วิดท์น้อยมาก

3. Node-RED (Flow-based Programming)
คืออะไร: เครื่องมือเขียนโปรแกรมแบบ Low-code จาก IBM ที่ใช้การลากเส้นเชื่อมต่อโหนด (Node)
บทบาทในโปรเจกต์: เปรียบเสมือนสมองของระบบ ทำหน้าที่หยิบข้อมูลดิบจาก Mosquitto มาทำความสะอาด (เช่น แปลง String เป็น Number), คำนวณค่า, หรือสร้างเงื่อนไข (เช่น ถ้าอุณหภูมิสูงเกินไปให้ส่งคำสั่งกลับไปเปิดพัดลม) ก่อนจะส่งข้อมูลที่สมบูรณ์ไปเก็บในฐานข้อมูล

4. InfluxDB (Time-Series Database)
คืออะไร: ฐานข้อมูลที่ออกแบบมาเพื่อเก็บข้อมูลที่มีแกนเวลา (Time-series) เข้ามาเกี่ยวข้องโดยเฉพาะ
บทบาทในโปรเจกต์: ใช้เก็บบันทึกประวัติข้อมูลเซ็นเซอร์ทั้งหมด ข้อดีคือสามารถรองรับการเขียนข้อมูลความถี่สูงได้ดีกว่าฐานข้อมูลทั่วไปอย่าง MySQL และมีระบบจัดการลดขนาดข้อมูลเก่า (Retention Policy) อัตโนมัติ

5. Grafana (Data Visualization)
คืออะไร: แพลตฟอร์มสำหรับสร้าง Dashboard ระดับองค์กร
บทบาทในโปรเจกต์: ดึงข้อมูลที่เก็บไว้ใน InfluxDB มาวาดเป็นกราฟ, เกจวัด, หรือตารางที่สวยงาม เพื่อให้ผู้ใช้งาน (User/Admin) สามารถวิเคราะห์แนวโน้ม (Trend) ของข้อมูลได้อย่างรวดเร็ว


🧠 เทคนิคและแนวคิดที่นำมาใช้ (Techniques Applied)

1. Microservices Architecture: แบ่งการทำงานออกเป็นส่วนๆ (รับข้อมูล, ประมวลผล, เก็บข้อมูล, แสดงผล) ทำให้หากมี Service ใดตัวหนึ่งล่ม (เช่น หน้าเว็บ Grafana ค้าง) ตัวเซ็นเซอร์ก็ยังสามารถส่งข้อมูลเข้า Broker และ Node-RED ได้ตามปกติ ระบบไม่ล่มทั้งระบบ

2. Publish-Subscribe Pattern (Pub/Sub):
ใช้เทคนิค Pub/Sub ผ่าน MQTT ซึ่งช่วยลดภาระของเซ็นเซอร์ เซ็นเซอร์มีหน้าที่แค่ "ตะโกน" (Publish) ข้อมูลออกมาตรงกลาง โดยไม่ต้องสนใจว่าจะมีใครรอรับอยู่หรือไม่ ทำให้เซ็นเซอร์ทำงานเบาลงและประหยัดแบตเตอรี่

3. Data Transformation & Sanitization:
ใช้ Node-RED ในการดักจับและแปลงประเภทข้อมูล (เช่น บังคับให้เป็น Float/Integer) ก่อนนำเข้า InfluxDB เพื่อป้องกันปัญหา Data Type Conflict ซึ่งเป็นปัญหาคลาสสิกในการทำฐานข้อมูล

4. Infrastructure as Code (IaC):
ใช้ไฟล์ docker-compose.yml เพียงไฟล์เดียวในการอธิบายและสร้างโครงสร้างเซิร์ฟเวอร์ทั้งหมด ทำให้สามารถนำโปรเจกต์นี้ไปติดตั้งใหม่บน Cloud, Raspberry Pi หรือ Local Server ได้ภายในเวลาไม่กี่นาที
